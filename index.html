01
Umumiy Ma'lumot

Smartup — savdo agentlari, sotish bo'limi, buxgalteriya, ombor va logistika ishlarini avtomatlashtirish uchun ERP-tizim. Distribyutsiya, trade marketing, ulgurji va chakana savdoni boshqarish uchun bulutli yechim.

Smartup API'lari tashqi tizimlar bilan integratsiya yechimlarini yaratish uchun mo'ljallangan. Hujjat tashqi tizimlar bilan ma'lumot almashishda ishlatiladigan ma'lumotlar modeli va API ishlash tamoyillarini batafsil tavsiflaydi.

Xususiyat Qiymat Izoh
Protokol HTTPS HTTP over SSL — nisbatan xavfsiz
Format JSON / XML Ikkala format ham qo'llab-quvvatlanadi
Encoding UTF-8 Barcha so'rovlarda majburiy
Auth Basic Auth base64(username:password)
HTTP metodlar GET, POST, PUT, DELETE So'rov va javoblar HTTP metodlariga asoslangan
Base URL https://smartup.online Barcha so'rovlar uchun asosiy manzil
Asosiy Kontsepsiyalar
Atama Tavsif
document_id movement_id, stocktaking_id, writeoff_id, return_id, input_id, deal_id — ularning umumiy nomi
external_id Tashqi tizim tomonidan berilgan obyekt/yozuv identifikatori
filial_code Tashkilotning unikal kodi
inventory_kind P — mahsulotlar, G — tovarlar, M — xom ashyo
state A — faol (Active), P — nofaol (Passive)

03
Autentifikatsiya

Smartup tizimida avtorizatsiya Basic Auth prinsipiga ko'ra amalga oshiriladi. Login va parol Authorization headerida base64 formatida kodlanib yuboriladi.

HTTPS protokoli bilan birga ishlatilganda bu nisbatan xavfsiz usul hisoblanadi.

Authorization Parametrlari
Parametr Tur Majburiy Tavsif
username string YES Smartup X tizimiga kirish uchun login
password string YES Smartup X tizimiga kirish uchun parol
base64 kodlash misoli
# Linux / Mac terminalida:
echo -n "myuser:mypassword" | base64
# Natija: bXl1c2VyOm15cGFzc3dvcmQ=

# cURL so'rovida ishlatish:
curl --location 'https://smartup.online/b/...' \
  -H "Authorization: Basic bXl1c2VyOm15cGFzc3dvcmQ=" \
  -H "project_code: trade" \
  -H "filial_id: 1" \
  -H "Content-Type: application/json" \
  --data '{ ... }'
Tashkilot Turlari
Tur Tavsif
Single organization Barcha tegishli entitylar sozlanishi mumkin bo'lgan tashkilot. Boshqa yagona tashkilotlarga kirish imkoni yo'q.
Head organization Ma'muriy tashkilot — barcha yagona tashkilotlarni boshqarish imkoniyati bor.

04
Majburiy Headerlar
ℹ️
project_code va filial_id — barcha so'rovlarda yuborilishi tavsiya etiladi. Ular qaysi loyiha va tashkilot uchun so'rov bajarilishini belgilaydi.
Header Tavsif Misol
Authorization Basic Auth — base64 kodlangan login:parol Basic dXNlcjpwYXNz
project_code Loyiha identifikatori. Qaysi loyiha bilan ishlashingizni belgilaydi trade / anor / clinic / telecom
filial_id Single yoki Head Organization ID si 1
Content-Type So'rov formatini belgilaydi application/json
filial_id qoidalari
Holat Natija
Single org filial_id Barcha so'rovlar faqat tanlangan tashkilot bo'yicha
Head org filial_id So'rovlar filial_code parametri asosida bajariladi
filial_code ko'rsatilmasa Barcha tashkilotlar bo'yicha ma'lumot ko'rsatiladi
